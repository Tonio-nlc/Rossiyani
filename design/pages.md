# ROSSIYANI — PAGE SPECIFICATIONS

Version 2.0

This document defines the architecture, hierarchy and UX principles of every major page.

The goal is consistency.

Every page should feel like a chapter of the same editorial product.

Never design pages independently.

Every page belongs to the same knowledge system.

---

# GLOBAL RULES

Every page follows this structure:

Large editorial title

↓

Small introduction

↓

Primary content

↓

Knowledge relationships

↓

Examples

↓

Related concepts

↓

References

↓

Whitespace

Never build dashboard layouts.

Never fill empty space.

Never create visual noise.

---

# HOME

## Purpose

The home page is an entry point.

It is not a dashboard.

It is not an analytics page.

It should immediately invite exploration.

The user should feel:

# PAGE CHECKLIST

Before validating any page ask:

Does it look like software?

If yes,

simplify.

Does it feel like an editorial linguistic explorer?

If yes,

keep it.

Does typography dominate?

Does whitespace breathe?

Are relationships visible?

Is exploration encouraged?

If every answer is yes,

the page respects the Rossiyani identity.
