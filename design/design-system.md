# ROSSIYANI — DESIGN SYSTEM

Version 2.0

This document is the canonical visual reference for Rossiyani.

If any existing implementation conflicts with this document,
this document always wins.

Rossiyani is an editorial product.

Never design it as a SaaS application.

---

# 1. DESIGN PRINCIPLES

Every visual decision must improve:

reading

understanding

orientation

memory

relationships

Never optimize for visual effects.

Always optimize for comprehension.

Typography is Rossiyani's primary identity.

Hierarchy comes from typography.

Not from containers.

8px

Whitespace creates hierarchy.

Never compress content.

---

# 7. LAYOUT

Maximum content width

1200px

Reading width

700–760px

Side annotations

280–340px

Left navigation

260–300px

Always preserve generous margins.

---

# 8. GRID

Use editorial rhythm.

Not dashboard grids.

Prefer:

sections

columns

reading flow

references

over equal cards.

---

# 9. SEPARATORS

Default:

1px hairline

soft neutral

full width

Never use heavy borders.

Never use thick outlines.

---

# 10. CARDS

Cards are discouraged.

Preferred alternative:

editorial sections

If cards are necessary:

flat

paper colored

1px border

24px padding

no shadow

no glow

no gradient

---

# 11. BUTTONS

Buttons should almost disappear.

Primary

filled

editorial

rounded 999px

minimal

Secondary

outlined

quiet

Text button

preferred whenever possible

Never oversized.

Never aggressive.

---

# 12. INPUTS

Flat.

Minimal.

Paper colored.

Thin border.

Large padding.

Large placeholder.

Feels like writing in a notebook.

---

# 13. SEARCH

Search is central.

Always prominent.

Large horizontal field.

Minimal border.

No colorful backgrounds.

Feels like searching an encyclopedia.

---

# 14. TAGS

Prefer editorial metadata.

Instead of:

[VERB]

[TRANSITIVE]

Use:

Verb · transitive · imperfective

Tags should feel printed.

Not interactive chips.

---

# 15. ICONOGRAPHY

Thin stroke.

Minimal.

Editorial.

Functional.

No filled icons.

No decorative illustrations.

---

---

# 19. READER

The reader is sacred.

Maximum comfort.

Large line-height.

Very large Russian text.

Translations feel like annotations.

Grammar feels like notes.

No visual distraction.

---

# 20. EXPLORER

Explorer should feel like browsing an encyclopedia.

Never like filtering a database.

Discovery first.

Search second.

---

# 21. MOTION

Allowed

opacity

slide

height expansion

Forbidden

bounce

zoom

elastic

glow

rotation

parallax

Animation should almost disappear.

---

# 22. SHADOWS

Avoid shadows completely.

Prefer:

spacing

contrast

hairlines

paper hierarchy

---

# 23. BORDER RADIUS

---

# 24. IMAGES

Very few.

Prefer:

diagrams

tables

references

annotations

Rossiyani is text-first.

---

# 25. COMPONENT PHILOSOPHY

Every component must answer:

Does it improve understanding?

If not,

remove it.

---

# 26. RECOGNIZABLE SIGNATURE

Rossiyani should always contain:

warm paper background

large editorial serif title

extreme whitespace

hairline separators

semantic colors

margin annotations

knowledge diagrams

minimal UI

If one of these disappears,

the identity becomes weaker.

---

# 27. THINGS NEVER TO BUILD

Glassmorphism

Neomorphism

Heavy gradients

Dashboard widgets

Floating cards

AI chat aesthetics

Startup landing pages

Gamification

Confetti

Score systems

Progress rings everywhere

Complex illustrations

---

# 28. FINAL RULE

Rossiyani should always look closer to

a beautifully designed encyclopedia

than

a beautifully designed web application.
