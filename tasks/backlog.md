# Règles

Avant de développer une fonctionnalité, vérifier que :

* elle respecte le Product Bible ;
* elle améliore réellement l'expérience utilisateur ;
* elle reste cohérente avec la philosophie du produit ;
* elle réutilise le Knowledge Graph lorsque c'est possible ;
* elle n'ajoute pas de complexité inutile.

La priorité est toujours de terminer les fonctionnalités existantes avant d'en créer de nouvelles.
