# Reader

## Objectif

Le Reader est le cœur de Rossiyani.

Il permet à l'utilisateur de lire des textes russes authentiques dans un environnement conçu pour favoriser la compréhension, la progression et le plaisir de lire.

Chaque phrase devient interactive et chaque mot peut être exploré sans interrompre la lecture.

Le Reader n'est pas un simple lecteur de texte : c'est l'expérience centrale autour de laquelle tout le produit est construit.

---

# Objectifs pédagogiques

Le Reader doit permettre à l'utilisateur de :

* comprendre des textes russes authentiques ;
* apprendre du vocabulaire en contexte ;
* comprendre les mécanismes grammaticaux ;
* développer progressivement son autonomie en lecture ;
* relire facilement les textes étudiés.

---

# Fonctionnalités principales

## Lecture

* Lecture confortable.
* Typographie optimisée.
* Mise en page éditoriale.
* Navigation simple entre les textes.

---

## Audio

Chaque texte et chaque phrase peuvent être écoutés.

L'utilisateur peut ralentir la vitesse de lecture sans dégrader la qualité de la voix.

---

## Traductions

Les traductions restent optionnelles.

L'utilisateur peut :

* masquer toutes les traductions ;
* afficher uniquement certaines phrases ;
* afficher toutes les traductions.

Rossiyani encourage progressivement une lecture de plus en plus autonome.

---

## Analyse linguistique

Chaque phrase possède une analyse pédagogique.

Les explications doivent être courtes, accessibles et centrées sur la compréhension.

Le vocabulaire, la grammaire et les constructions sont toujours expliqués dans leur contexte.

---

## Interaction avec les mots

Un clic sur un mot ouvre sa fiche Vocabulary.

L'utilisateur peut :

* écouter le mot ;
* consulter sa définition ;
* voir ses formes ;
* l'ajouter aux révisions ;
* accéder aux exemples.

La lecture ne doit jamais être interrompue.

---

## Progression

Le Reader mémorise automatiquement :

* les textes lus ;
* le temps de lecture ;
* les mots rencontrés ;
* les mots révisés ;
* la progression de lecture.

---

## Recherche

Une recherche intégrée permet de retrouver rapidement un mot ou une expression à l'intérieur du texte.

---

# Expérience utilisateur

Le Reader doit donner la sensation de lire un vrai livre.

Les outils apparaissent uniquement lorsqu'ils sont utiles.

L'interface reste calme, minimaliste et centrée sur le contenu.

Chaque interaction doit être instantanée.

---

# Version mobile

Le Reader mobile est pensé pour une utilisation à une main.

Les informations complémentaires apparaissent sous forme de panneau ou de feuille coulissante afin de conserver toute la place nécessaire au texte.

La lecture reste toujours prioritaire.

---

# Données utilisées

Le Reader s'appuie principalement sur :

* la bibliothèque de textes ;
* le Knowledge Graph ;
* les analyses linguistiques ;
* les traductions ;
* les fichiers audio ;
* la progression utilisateur.

---

# Évolutions futures

Le Reader pourra intégrer progressivement :

* des exercices générés à partir du texte ;
* des recommandations de lecture personnalisées ;
* des annotations personnelles ;
* des favoris ;
* un mode lecture immersive ;
* un mode hors ligne.

Toutes les évolutions devront respecter un principe fondamental : ne jamais perturber l'expérience de lecture.
